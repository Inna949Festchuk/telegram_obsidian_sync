## ⚠️ Где потребуются доработки человека

| Проблема | Почему AI может ошибиться | Что делать |
|----------|-------------------------|------------|
| **Unicode пути** | Русские символы в путях (`/home/konstantin/Загрузки/...`) | Тестировать на реальных путях |
| **Индексация медиа** | Логика `patch.txt` парсинга сложная | Проверить вручную |
| **Коллизии имён** | Дубликаты файлов в разных чатах | Добавить тесты |
| **Большие JSON** | Файлы 100MB+ могут вызвать проблемы | Добавить потоковую обработку |
| **Edge cases** | Пустые чаты, битые файлы, missing keys | Добавить валидацию |
| **Производительность** | AI не оптимизирует автоматически | Профилировать и оптимизировать |
| **Тесты** | AI пишет тесты формально | Написать интеграционные тесты |

## 🔧 Рекомендации для работы с AI-агентом

### 1. Разбейте задачу на итерации

```
Итерация 1: Базовый парсинг JSON → Markdown (без медиа)
Итерация 2: Добавление копирования медиа
Итерация 3: Индексация через patch.txt
Итерация 4: Обработка ошибок и edge cases
Итерация 5: Тесты и документация
```

### 2. Предоставьте контекст

```
- Приложите пример result.json (фрагмент)
- Приложите пример patch.txt
- Укажите точные пути к файлам
- Покажите ожидаемую структуру вывода
```

### 3. Требуйте тесты

```
"Напиши unit-тесты для метода find_media_file"
"Создай интеграционный тест с реальным экспортом"
```

### 4. Проверяйте критические части

```python
# Обязательно проверьте вручную:
- Кодировка файлов (UTF-8)
- Обработка русских путей
- Копирование больших файлов
- Обработка пустых чатов
```

## 🎯 План работы с AI-агентом

### Этап 1: Подготовка (1 день)
```
- Подготовьте CLOUD.md
- Подготовьте тестовый экспорт Telegram (небольшой)
- Создайте репозиторий на GitHub
```

### Этап 2: Базовая реализация (2-3 дня)
```
- AI генерирует основную структуру
- Реализует парсинг JSON
- Создаёт Markdown файлы
- Тестируете вручную
```

### Этап 3: Медиа и индексация (2-3 дня)
```
- AI добавляет копирование файлов
- Реализует patch.txt парсинг
- Тестируете с реальными медиа
```

### Этап 4: Полировка (1-2 дня)
```
- Обработка ошибок
- Логирование
- Документация
- Финальное тестирование
```

**Итого: 5-8 дней с AI-агентом** (vs 2-3 недели вручную)

## 💡 Моя рекомендация

**Используйте гибридный подход:**

```
1. AI генерирует 70-80% кода (структура, базовая логика)
2. Человек дорабатывает 20-30% (Unicode, edge cases, тесты)
3. AI помогает с документацией и рефакторингом
4. Человек проводит финальное тестирование
```

**Это даст лучший баланс скорости и качества.**
